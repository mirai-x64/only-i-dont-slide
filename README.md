# only-i-dont-slide / 私だけ滑らない

傾けても自分の玉だけは動かない。用紙も、他の受験者の玉も鉛筆も、こちらへ滑ってくる。

遊ぶ: https://mirai-x64.github.io/only-i-dont-slide/
(単一 HTML + Canvas。依存なし。ファイルを直接開いても動く)

## 操作

方向キーで巨大な解答用紙が傾く。自分の玉は画面の中央に貼り付いたまま動かない。
二重丸で囲まれた自分の解答欄 θ を、自分の真下まで引き寄せればクリア。

## 二種類の動き方

用紙と壁は**傾いた向きへ素直に平行移動する**。加速も慣性もなく、キーを離せば即止まる。
他の受験者の玉と鉛筆は**別物**で、傾きを加速度として受ける。キーを離しても慣性で
滑り続け、壁で跳ね返り、切り取り線から用紙の外へ落ちる。
だから両者が一体で動くことは無く、座標変換では消せない。

自分の玉に他の玉が触れると貼り付きが剥がれ、自分も慣性を持つただの玉として滑り出す。
そうなったら用紙の外か、他人の解答欄へ落ちるまでのわずかな猶予しかない。

## 1面が迫る選択

記入欄は横一本の細い通路で、**玉が二つ横に並べる幅が無い**(通路 46px に対し
自分の半径 13 + 相手の半径 12 = 25。すれ違うには 50px 要る)。
そこで待っている一人と正面から出会うと、避ける余地は無い。

自分の欄 θ は東にある。引き寄せる傾きは「左」だけ。
ところが左へ傾けるほど、通路の相手はこちらへ加速してくる。
実測すると、出会いの瞬間(約 0.9〜1.4 秒)には:

- 左成分を持つ傾きは**すべて死ぬ**
- 生き残る傾きは右成分を持つ三つだけで、目標までの距離は **-403〜-408px**(後退)

引き寄せる傾きと逃げる傾きが、この一瞬だけ両立しない。

抜け道は記入欄の床に一箇所だけ開いた切り取り線で、相手をその真上まで運んでから
下へ傾け続けると、用紙の外へ落ちていく。ただし下への傾きは自分の欄を 1px も
引き寄せない。自動操縦での最短は約 9.0 秒。

## 検証

`window.SIMAPI` に描画から独立した物理(`newState` / `step`)を出してある。
ヘッドレスブラウザから同じ物理を回して、上の数字と解の存在を確かめている。
